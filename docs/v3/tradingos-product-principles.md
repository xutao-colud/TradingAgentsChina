# A股 TradingOS 产品原则

## 一句话定位

TradingAgentsChina 不是股票预测机器人，而是一个会积累证据、记忆用户打法、复盘自身判断、持续优化策略的 A 股投资研究操作系统。

## 必须坚持

- 先判断市场生态，再分析个股。
- 先列证据和反证，再给结论。
- 先保护用户交易体系，再追求报告好看。
- 先做可复盘闭环，再增加 Agent 数量。
- 先做用户记忆、策略记忆和 Agent 信誉，再谈模型训练。

## 禁止方向

- 不承诺收益。
- 不输出自动交易指令。
- 不把 LLM 当实时行情源。
- 不用固定投票替代证据审判。
- 不做无法追溯来源的强结论。

## 当前落地机制

| 原则 | 当前代码落点 |
| --- | --- |
| 证据优先 | `证据链完整性` Skill 检查 evidence/source/time/counterpoint |
| 用户优先 | `TradingProfile` 与 `个人交易画像适配` |
| 策略优先 | Playbook 适配与 `投资流派委员会` |
| 记忆优先 | `LocalMemoryStore` 记录分析、反馈、问答摘要 |
| SaaS 谨慎聚合 | `StrategyOutcomeRecord` 只做授权后的观察性统计 |

## 后续优先级

1. Agent Reputation：按市场状态统计每类 Agent 历史表现。
2. Strategy Regime Analytics：同一战法按牛市、震荡、退潮、冰点分组。
3. Similar Case Memory：检索历史相似行情和失败原因。
4. Trading Coach：定期输出用户优势、弱点和亏损来源。
5. Knowledge Graph：政策 → 行业 → 产业链 → 公司 → 资金 → 历史案例。

## 输出标准

每份报告必须回答：

- 当前市场状态是什么？
- 哪条策略路线最有说服力？
- 支持证据是什么？
- 反对证据是什么？
- 证据链是否完整？
- 风险和不确定性在哪里？
- 为什么适合或不适合当前用户？
- 下一步应该观察什么？
