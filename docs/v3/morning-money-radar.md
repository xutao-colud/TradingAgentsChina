# 盘中资金雷达：数据边界与降级链路

## 目的

盘中资金雷达用于研究，不用于预测或下达交易指令。每次响应都包含 `source`、`data_status`、`as_of`、风险提示和不可用原因；界面必须按这些字段解释数据。

## 数据源优先级

1. `eastmoney_push2`：东方财富全市场盘中板块主力净流入/流出及个股涨速。仅 `data_status: real_time` 时可称为盘中数据。
2. `tushare_moneyflow_ind_ths`：主源失败后的受权限保护备选。使用 Tushare `moneyflow_ind_ths` 返回的最近完整交易日行业资金流，状态恒为 `latest_available`。它是盘后数据，**不能**推断当前盘中板块资金、涨速或盘口。
3. `sina_tracked_universe`：前两层不可用时，仅展示自选、持仓和机会池的可核验报价。它不代表全市场，并且不提供板块资金流、主力净流入或涨速。
4. `unavailable`：没有取得可核验事实时，不展示样例板块或样例个股，也不生成短线方向判断。

AkShare 的 `*_em` 适配器与东方财富链路相关，不能被标记为独立的全市场盘中备源。

## 启用盘后行业资金备源

Tushare 的 `moneyflow_ind_ths` 文档明确说明该接口为每日盘后更新，并要求相应账户权限。将 Token 配置在**运行服务的机器**上后重启服务：

```powershell
$env:TUSHARE_TOKEN = "你的Token"  # 仅当前 PowerShell 会话
python -m app.web.server --host 0.0.0.0 --port 8000
```

要持久化请使用部署密钥管理或系统环境变量；不要把 Token 写进仓库、浏览器或 LAN 页面。没有 Token 或接口权限时，系统会继续走跟踪池/不可用降级，不会补造行业数据。

## 使用约束

- 盘后行业资金只可用于复盘、跨日观察和后续研究的候选池排序。
- 跟踪池报价只可描述覆盖范围内的个股，不可概括成“市场主线”。
- 即使主源实时可用，板块资金仍需与市场状态、龙头反馈、公告、流动性和 TradingProfile 交叉核验。
- 所有源不可用时，正确结果是“证据不足”。
