# 投资流派委员会

## 定位

`投资流派委员会` 是 A 股 TradingOS 的路线选择层。它不直接判断“买/卖”，而是在同一份确定性报告上比较多种市场打法，输出当前更有说服力的研究路线。

核心目标：

- 把激进、稳健、价值、防守等流派放到同一证据表里比较；
- 让报告说明“为什么这条路更适合当前环境”；
- 在市场退潮、规则否决或风险升高时，让防守路线能够胜出；
- 为 SaaS 版本沉淀用户战法、盈亏反馈和群体统计留出接口。

## 当前覆盖流派

| 流派 | 路线 | 更适合的市场 |
| --- | --- | --- |
| 激进游资派 | 情绪龙头/题材接力 | 情绪发酵、赚钱效应强、资金连续确认 |
| 趋势容量派 | 趋势确认/回踩低吸 | 趋势未破、资金温和流入、题材扩散 |
| 机构成长派 | 景气成长/预期修正 | 基本面强、风险可控、估值有业绩支撑 |
| 价值红利派 | 现金流/估值约束 | 财务质量高、估值不过度透支、波动可承受 |
| 政策周期派 | 产业政策/板块轮动 | 政策主题清晰、资金扩散、市场温度可用 |
| 低吸反转派 | 冰点修复/恐慌低吸 | 情绪冰点或退潮后出现修复确认 |
| 防守风控派 | 仓位控制/等待确认 | 风险扫描不足、规则约束、情绪退潮或冰点 |

## 输入证据

委员会只读取已经计算出的 Agent 与 Skill 结果：

- Agent：市场、基本面、技术、资金、公告、题材；
- Skills：市场温度、情绪周期、赚钱效应、题材生命周期、主力资金行为、A 股风险扫描、综合评分；
- A 股规则：ST、停牌、流动性、换手率等否决/降级条件。

它不让 LLM 自己猜行情，也不依赖未验证的用户传闻。

## 输出解释

报告会新增一个 `SkillInsight`：

```json
{
  "skill": "投资流派委员会",
  "category": "committee",
  "stage": "趋势容量派",
  "score": 92,
  "conclusion": "当前证据对趋势容量派的适配度明显领先。",
  "strategy": "优先等待回踩不破、成交额确认和资金持续性，避免在加速段追价。"
}
```

这里的 `score` 是证据适配度评分，不是收益承诺、预测概率或胜率。它用于产品内的路线选择与复盘分组；真实 SaaS 版本仍必须接入用户授权的结果反馈、交易成本、市场阶段和样本外验证。

## 法庭流程

委员会不投票。每条路线必须提交支持证据、反对证据和失效条件；随后输出逐路线 `cross_examination`、全局 `risk_challenge`，最后才由 judge 汇总。任何规则约束、证据链缺口或失效条件缺失，都使结论降级为观察，不构成行动指令。

## SaaS 迭代方向

后续可以把每次委员会输出与用户复盘结果关联：

```text
报告ID
流派胜出结果
用户选择的战法
持有周期
实际收益
市场阶段
是否授权匿名聚合
```

聚合层只回答观察性问题，例如：

> 在已授权、样本量足够的记录中，趋势容量派在震荡修复环境下的适配分数与后续收益存在正相关。

不能回答：

> 趋势容量派一定有效，或某个用户/某个流派稳定跑赢。

这保证产品定位仍是投资研究辅助和交易系统优化，而不是收益承诺工具。
