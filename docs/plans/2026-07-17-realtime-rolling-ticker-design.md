# 实时报价滚动设计

## 方案

新增报价专用接口，仅拉取自选股和本地持仓的价格字段。完整行情刷新继续负责板块、概念和资金分档，避免每三秒重复请求昂贵数据。

前端使用可配置的单次定时器轮询：页面可见且存在跟踪股票时请求；请求未完成不重入；失败后按配置退避；页面恢复可见时立即补一次。实时交易状态约三秒刷新，最近可用/非实时状态自动降低频率。价格相对上一笔上涨时整体数字向上滚并显示 A 股红色，下跌时向下滚并显示绿色，不变时不播放动画。

## 数据边界

- 东方财富轻量报价优先，缺失股票批量回退新浪报价。
- 数据状态、来源、交易日期与时间随报价返回。
- 报告中的实时证据快照保持冻结，不随页面滚动改写。
- 轮询间隔、失败退避、动画时长和最大股票数均来自运行配置。

## 验证

- 报价接口只访问报价 URL，不访问资金流 URL。
- 多数据源降级不覆盖已经可用的报价。
- 页面隐藏、空列表和请求进行中不产生新请求。
- 上涨、下跌、不变、不可用四种 UI 状态均可测试。
