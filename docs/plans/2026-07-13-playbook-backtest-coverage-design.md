# 策略回测覆盖范围扩展设计

## 决策

保留现有价格回测引擎，在规则外增加 `PointInTimeDataset`。规则仍只在每日收盘后形成信号、下一交易日开盘撮合，但可查询“截至该日已经公开”的市场、题材、个股、财务、预期、估值和分红记录。

不采用预先生成的布尔信号，因为它会隐藏信号使用了哪些原始时点；不在本轮重写通用回测引擎，因为现有 T+1、涨跌停、流动性、手续费、印花税和滑点约束已经可用。

## 时点化数据契约

- `MarketBacktestObservation`：观察日情绪周期、涨跌停、炸板率。
- `ThemeBacktestObservation`：题材在观察日的生命周期和强度。
- `ThemeMembershipObservation`：成分关系的生效区间及系统知晓时间，防止用当前成分回填历史。
- `StockBehaviorObservation`：个股涨停状态、连板、资金连续性和题材核心排序。
- `FundamentalBacktestObservation`：报告期、公告日、三表派生指标和盈利质量；只有公告日之后可见。
- `ConsensusBacktestObservation`：一致预期观察日和修正幅度。
- `ValuationBacktestObservation`：历史 PE/PB。
- `DividendBacktestObservation`：分红公告日、股息率和派息约束。

所有记录必须有可用时间和来源标识。报告期不得晚于公告日；成分关系同时受生效时间和知晓时间约束。查询接口只返回 `available_at <= as_of` 的最新记录。

## 战法规格

### hot_money_leader

在情绪启动/发酵、涨停扩散、炸板率受控、题材处于启动/扩散、成分关系当时已知、核心排序和多日资金满足配置时形成信号；退潮、题材退潮或连板过热退出/拒绝。

### institutional_growth

使用公告时点财务、盈利增速、ROE、现金流质量、历史估值和一致预期修正；财务或预期恶化、估值越界或趋势破坏退出。

### institutional_value_dividend

使用公告时点三表、现金流质量、杠杆、历史 PE/PB、分红公告、股息率和派息率；现金流恶化、杠杆上升、分红不可持续或估值越界退出。

## 门禁与验证

- 每种战法在构建规格前检查必需数据类型、来源和时间覆盖；缺失时明确拒绝。
- 不以少量样本展示正收益比例，沿用最少 30 笔门槛。
- 回测继续支持标准滑点和压力滑点；结果按市场状态拆分。
- 测试主动放入未来公告、未来成分和未来预期，验证规则在公告前不可见。
