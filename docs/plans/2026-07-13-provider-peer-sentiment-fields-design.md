# Provider 同业基准与情绪历史字段改造

## 目标

消除 `FundamentalSnapshot.peer_medians` 的空字段，并确认 `MarketContext.sentiment_history` 由连续多日真实市场观察填充。任何缺失、权限失败或期次不一致都必须显式降级，不能生成中性样例值。

## 数据流

- 工作流把 `analysis_date` 传给基本面 Provider，财务记录仅使用该日期前已公告的数据。
- Tushare 股票列表确定目标行业和同行代码。
- 优先通过 `fina_indicator_vip(period=报告期)` 批量拉取同行数据；权限不可用时，按配置上限逐股调用 `fina_indicator`。
- 同行记录必须与目标财报期一致，并通过样本量、日期、唯一性和数值质量校验后才计算中位数。
- 同业样本量、报告期、不可用原因、证据来源和原始快照随 `FundamentalSnapshot` 保存。
- 情绪历史继续使用指数交易日确定窗口，逐日组合全市场日线和涨跌停池；只有连续且完整的观察才参与速度与加速度计算。

## 失败策略

- 找不到行业、目标报告期或足够同行样本时，`peer_medians` 保持为空并记录原因。
- 批量接口权限不足时允许配置化的有限逐股回退，但不得超过请求上限。
- 情绪历史任一交易日缺少市场宽度或涨跌停池时，质量报告失败，市场状态不得标记为已核验。
