# 可审计证据简报设计

## 目标

提高报告说服力，但不依赖模型写得更长。报告首先用确定性代码回答：结论由哪些证据决定、最强反证是什么、何时失效、委员会为什么选择当前路线、结论是否符合用户画像，以及还缺哪些关键数据。

## 数据契约

`AnalysisReport.decision_brief` 包含：

- 结论主张与数据边界；
- 带来源、时间、分数和置信度的决定性证据；
- 与当前证据方向相反的最强反证；
- 可观察的失效条件；
- 法庭式委员会胜出路线、第二路线和领先差；
- TradingProfile 适配结论；
- 去重、限量后的关键数据缺口。

所有显示数量与截断长度来自运行配置，不散落硬编码。没有来源的事实不能进入决定性证据。

## 模型边界

模型输入改为精简证据包，不再发送完整质量报告和全部 Skills。模型可以解释和组织，但不得改变确定性分数、风控、来源与委员会裁决。未配置模型时，确定性证据简报仍完整展示。

## 前端

在分数之后优先展示“证据裁决简报”，包括主张、决定性证据、反证/失效条件、委员会差异、画像适配和关键缺口。详细 Skills 和完整法庭继续保留，作为下钻材料。
