# 盘中雷达报价容灾设计

## 问题

东方财富全市场板块列表不可达时，雷达原本只调用新浪报价作为跟踪池降级源，绕过了实时行情已经具备的腾讯/新浪容灾链。因此新浪失败后，页面错误地退化为“无可核验数据”，即使腾讯仍能返回当日报价。

## 改造

- 全市场雷达：仍以东方财富板块资金流为主，并保留已配置的东方财富备用域名与 Tushare 盘后行业资金源。
- 跟踪池雷达：复用 Web 行情统一链路，依次尝试东方财富个股快照及配置中的腾讯、新浪报价源。
- 数据门禁：只接纳实时状态或交易日期等于当前日期的正价格报价；拒绝历史缓存冒充盘中数据。
- 来源追踪：由实际成功的报价源动态生成 `tracked_universe:<provider>`，不再硬编码新浪。
- 口径隔离：跟踪池只展示自选、持仓和机会池异动；板块资金流、主力资金等缺失字段保持为空，不据此推断全市场。

## 验证

- 东方财富雷达失败且腾讯可用时，返回 `tracked_universe` 而非 `unavailable`。
- 前一交易日缓存不能进入盘中雷达。
- 跟踪池降级不生成板块资金流数据。
