# 行业景气度确定性链路设计

## 目标

把行业资金、估值、盈利和产业链传导从题材匹配中拆出，形成可追溯、可降级、可测试的行业景气度事实链。该能力只描述当前证据，不输出收益概率或交易指令。

## 数据契约

- `IndustryFlowObservation`：同日全行业资金净额、涨跌幅与公司数。
- `IndustryValuationObservation`：同行样本在一个观察日的 PE/PB 中位数与样本数。
- `IndustryChainNode`：由版本化配置声明的上游/中游/下游行业关系及知识来源。
- `IndustryContext`：保存目标行业、全行业资金、估值历史、产业链节点、来源和不可用原因。

Provider 只填充真实记录；权限不足、分类无法对齐或样本不足时保留 `unavailable_reasons`，不得补零。

## Tushare 实现

1. `moneyflow_ind_ths` 拉取分析日全行业资金流，原始快照落盘并进行日期、唯一性和数值校验。
2. 使用 `stock_basic` 动态取得同行证券，不在代码中维护 `industry_map`。
3. 用同行 `daily_basic` 历史数据构建每个观察日的 PE/PB 中位数；只采纳正值、有限值和达到最小样本量的数据点。
4. 盈利增速复用同报告期同行财务中位数，避免重复查询和口径漂移。
5. 产业链关系来自运行配置，节点的当日景气信号只能由已校验的行业资金记录支持；缺失节点明确标记为数据不足。

## Skill

`industry_prosperity.py` 确定性计算：

- 目标行业资金净额排名和百分位；
- 当前行业 PE/PB 在自身历史中的分位；
- 个股营收/利润增速相对行业中位数的差值；
- 上游/中游/下游节点资金方向是否一致，以及传导链是否完整。

评分只是“证据适配度”，所有权重、阈值、样本数要求均由配置提供。缺少目标行业映射或关键数据时输出“证据不足”。

## 集成

- 工作流先采集行业上下文，再运行行业景气 Skill。
- EvidenceSource 和 DataQualityReport 纳入统一证据链。
- 委员会仅在来源可追溯且质量通过时，将行业景气小权重用于机构成长派、价值派和政策周期派；不得覆盖公司财务和风险否决。

## 测试

- 资金排名和单位换算；
- PE/PB 历史分位；
- 个股与行业增速差；
- 完整/断裂产业链传导；
- 空数据、权限失败、行业无法匹配时拒绝强结论；
- 工作流与委员会接入回归。
