# A股公开风格原型库

## 定位

本库把公开可描述的研究框架沉淀为确定性规则，不复制或归因于任何具体游资、基金经理或机构的非公开方法。它用于研究排序、风控与复盘，不输出自动交易指令。

| ID | 原型 | 研究周期 | 最重要的否决条件 |
| --- | --- | --- | --- |
| `hot_money_leader` | 情绪龙头确认 | 1-5 个交易日 | 情绪退潮/冰点、赚钱效应弱、流动性不足 |
| `trend_core` | 趋势容量核心 | 5-20 个交易日 | 趋势破坏、资金背离、题材高潮负反馈 |
| `institutional_growth` | 机构景气成长 | 1-3 个月 | 盈利/预期恶化、估值无支撑、重大风险公告 |
| `institutional_value_dividend` | 机构价值红利 | 1-6 个月 | 现金流恶化、高杠杆、价值陷阱或纯题材驱动 |

每个原型都包含：适用市场、所需证据、否决条件、优化重点与可检验假设。报告输出的“公开风格原型适配”只说明当前条件是否匹配，不能把不适配的市场硬解释为机会。

## 投资流派委员会

除了用户主动选择的公开风格原型，报告会先运行市场状态策略门槛，再运行 `投资流派委员会`。它会同时比较激进游资派、趋势容量派、机构成长派、价值红利派、政策周期派、低吸反转派和防守风控派，并输出当前证据适配度最高的研究路线，不输出或暗示胜率。

这个委员会用于回答：

> 当前市场和个股证据下，哪一种打法更有说服力？

它不用于回答：

> 哪个派系永远最强，或下一笔交易一定盈利。

当情绪退潮、风险扫描偏低、ST/停牌/流动性等规则约束出现时，防守风控派可以成为胜出路线，并会压低最终报告结论等级。

## 切换

```powershell
python -m app.cli --list-playbooks
python -m app.cli --playbook trend_core
python -m app.cli --playbook institutional_growth
```

也可以在本地工作台的“当前公开风格原型”下拉框中切换。该选择保存在 `TradingProfile`，随导出 JSON 一起迁移。

## 进入实盘前的验证门槛

原型当前是研究假设，不是已验证收益策略。任何原型若要标记为“已验证”，至少应完成：

1. 明确股票池、入场、退出、仓位与 A 股交易限制；
2. 覆盖牛熊和高低波动环境的足够历史样本；
3. 加入佣金、滑点、涨跌停、停牌与无法成交等悲观摩擦；
4. 参数敏感性和样本外检验；
5. 按年度、市场状态和失败样本复盘，而非只展示成功案例。

## 回测数据覆盖

| 原型 | 回测输入 | 时点门禁 |
| --- | --- | --- |
| `trend_core` | 日线、成交量、流动性、市场状态 | 收盘形成信号，次日开盘撮合 |
| `hot_money_leader` | 历史情绪、涨跌停/炸板、题材生命周期、历史题材成分、个股核心排序和多日资金 | 题材成分同时满足生效日与系统知晓日，禁止用当前成分回填历史 |
| `institutional_growth` | 公告时点财务、历史一致预期、历史估值 | 财报只在公告日后可见，一致预期按观察日滚动 |
| `institutional_value_dividend` | 公告时点三表、历史估值、分红公告 | 三表和分红只在公开日后可见，禁止按报告期提前使用 |

后三类战法必须通过 `PointInTimeDataset` 构建规格；价格数据齐全但事件数据缺失时仍会拒绝回测。所有结果继续执行交易成本、压力滑点、T+1、流动性和涨跌停约束，并按市场状态拆分。样本不足时不显示经验正收益比例。
